Rate Limits
Each team with a paid API account is allowed a fixed number of requests per subscription period with a maximum burst of 20 requests per minute, spread over that minute.

We reserve the right to adjust the rate limit for given endpoints to provide a high quality of service for all API customers.

The following HTTP headers can be used to track usage.

 

HTTP Header
Description
Current
X-RateLimit-Limit
Request limit per subscription period
3000000
X-RateLimit-Remaining
The number of requests left for the subscription period
2967823
X-RateLimit-Reset
The timestamp of the next rate limit reset
2025-08-16 07:26:26 UTC

Sentiment
Sentiment will be positive, neutral, negative, mixed or nil. nil indicates sentiment has not been processed for this comment.

Page Depth Limit
You can retrieve up to 50,000 reviews using the page parameter. If you need to retrieve more (or all) reviews you can slice and iterate by date using the start and end parameters.

Android OS Version
os_version for Android provides the API level. See our support site on how to convert the API level to an Android version number.
